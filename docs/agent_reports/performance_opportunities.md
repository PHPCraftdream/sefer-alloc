# Read-only аудит производительности

## Статус

Аудит исходного кода не выполнен: в текущей сессии отсутствует отдельное read-only средство для чтения текстовых файлов рабочего каталога. Доступ к ним возможен только через выполнение команд или скриптов, тогда как условия задачи прямо запрещают запуск любых команд и скриптов, включая Git, тесты, сборки и вспомогательные команды.

Чтобы не нарушать это ограничение, не выполнялись команды, тесты, сборки, бенчмарки или скрипты; исходники и конфигурация не изменялись. Поэтому ниже намеренно нет неподтверждённых утверждений о горячих путях, сложности алгоритмов, копированиях, аллокациях, cache locality, атомиках, блокировках или contention конкретного проекта.

## Подтверждённые возможности ускорения

Подтверждённых по исходникам возможностей нет: без чтения файлов любые конкретные выводы были бы предположениями, а не аудитом.

## Что необходимо для полноценного аудита

Нужно предоставить содержимое исходников и конфигурации в контексте запроса либо разрешить исключительно read-only команды для перечисления и чтения файлов. После этого статический аудит сможет охватить:

- асимптотику алгоритмов и повторные обходы данных;
- лишние `clone`, `to_owned`, промежуточные `Vec`/`String`, reallocations и dynamic dispatch;
- contention, гранулярность блокировок, false sharing и порядки памяти атомиков;
- layout структур, cache locality, pointer chasing и выбор коллекций;
- наиболее вероятные горячие пути и цену проверок, ветвлений и преобразований;
- настройки профилей Cargo, feature flags и зависимости, влияющие на производительность.

Никаких рекомендаций по изменению кода без доказательств из конкретных файлов не предлагается.
