## CONFIRMED

### C1 — refill-окно: заявление X2 «in-magazine нога закрыта» ложно для окна пополнения

- `src/registry/heap_core.rs:1390` — предикат refill-дренажа начинается с
  `if k == c { return false; }`, оправдано комментарием «KEY INVARIANT:
  count[c]==0». Аргумент доказывает *borrow*-безопасность, но НЕ семантическое
  покрытие: блоки, вытянутые в `tcache.slots[c][0..filled]` **в ходе текущего
  refill**, предназначены магазину и невидимы предикату.
- `src/alloc_core/alloc_core.rs:1911–1957` (`refill_class_bump_impl`):
  сначала вычерпывается **freelist** текущего сегмента
  (`drain_freelist_batch`, строка ~1935), и только потом вызывается
  `find_segment_with_free_checked` (~1945), который дренирует ринги.
  Интерливинг: блок P на freelist + стейл-записка кросс-потокового
  double-free на P ещё в ринге → шаг 1 тянет P в `out` (bitmap→allocated) →
  ринг-дренаж видит записку, bitmap allocated, предикат false для класса c →
  P перелинкован во freelist → тот же `find` возвращает сегмент → P вытянут
  в `out` **второй раз**.
- **Исполняемый PoC** (ревьюер): интеграционный тест против
  `--features production` только на существующих dbg-хуках
  (`dbg_flush_all` + `dbg_push_to_ring` + alloc-цикл) →
  «P double-issued (2 times)» — коррупция freelist/магазина от одного
  кросс-потокового double-free, ровно класс, который X2 объявил
  нейтрализованным.
- Почему наш зелёный тест это не ловит:
  `drain_resident_xthread_double_free_no_corruption` дренирует через
  `HeapCore::dbg_drain_all_rings` (`heap_core.rs:1652–1661`), чей предикат
  проверяет **все** классы включая c — *более сильный* предикат, чем
  production-refill-путь.
- **Таксономическое следствие:** «REMAINING residual = re-issue-before-drain»
  (`heap_core.rs:953–961`, дизайн-док §8, CHANGELOG X2, IAI_BASELINE)
  неполно. Эта нога НЕ информационно-неразличима — блок в момент дренажа
  magazine-resident (в out-буфере); теорема §8 её не покрывает. Правильная
  таксономия — ТРИ ноги:
  1. in-magazine-at-drain — закрыта X2;
  2. **in-refill-out-buffer-at-drain — дыра, закрывается фиксом R1 (дёшево,
     без поколений);**
  3. re-issue-before-drain (блок у пользователя) — теорема §8, territory X7.
- Форма фикса (выбран вариант А): `refill_class_bump_impl` владеет `out` —
  на каждом вызове `find_segment_with_free_checked` оборачивать предикат:
  `is_in_magazine(ptr,k) || (k == class_idx && out[..filled].contains(ptr))`
  (реборроу `&out[..filled]` на время вызова find; NLL это допускает —
  мутабельный borrow `out` в этот момент не используется). Вариант Б
  (дренировать ринги ДО первого пула из freelist) отвергнут: добавляет дренаж
  на каждый refill даже при достаточном freelist → рост Ir на refill-пути,
  за который боролись в X2 (+30 принятые). Вариант А стоит ноль на
  single-thread пути (пустой ринг → ноль записок → ноль сканов).
- **Проверено лично (2026-07-06):** порядок freelist-first и слепота
  предиката к `cur[0..filled]` подтверждены чтением
  `refill_class_bump_impl` (строки 1934–1953) и замыкания в
  `refill_magazine_slow` (heap_core.rs:1381–1397). E1-латч не мешает:
  дренажи происходят только до латча, в ветке `!free_exhausted`; carve-блоки
  в дренажи этого refill не попадают (source order).

### C2 — стейл-док + production-мёртвый дубликат realloc

- `src/registry/heap_core.rs:1072–1101` (док-блок) и `:1114–1118` всё ещё
  говорят «delegate to `self.core.realloc`» — код давно не вызывает его
  (использует `try_realloc_inplace` + `HeapCore::alloc`).
- `AllocCore::realloc` (`alloc_core.rs:1511`) production-мёртв (grep:
  единственное упоминание `core.realloc` — сам стейл-док), но несёт полный
  дубликат OPT-F/OPT-G логики (`:1556–1640` vs `try_realloc_inplace`
  `:1662–1713`) — непомеченная угроза расхождения.
- `heap_core.rs:1174` `if p != ptr { stamp }` — мёртвая ветка (in-place
  всегда возвращает `ptr`); комментарий `:1167–1168` «no drain needed»
  противоречит дренажу, уже случившемуся на `:1154` (безвредно — семантика
  дренажа консервативно-корректна, но текст лжёт).
- MUST-1/A1 и #169-штамп **сохранены** (проверено ревьюером чтением):
  Large-классифицированный новый размер дренирует на `:1147–1156`;
  alloc-нога идёт через `HeapCore::alloc` (дренаж + штамп).
