## 2. Что ревью C добавило нового (и куда это легло)

1. **§1.1 — методология headline (уточнение, самое важное):**
   `ns/owner_alloc` — ручной тайминг под-окна (`r12_7...wallclock.rs:181`,
   старт ПОСЛЕ pre-аллокации 6400 блоков, стоп до `recycle`); criterion
   меряет полный раунд. Из тех же raw-логов: N=8 full-round 20.6→18.4 ms
   (~11%), N=4 1.840→1.811 ms (~1.6%). ~17 ms drain-работы в treatment
   переехало в неизмеряемую часть раунда, а не исчезло. Ни один gate-док не
   приводит full-round рядом с headline. → **влито в #288** (печатать обе
   оси в харнессе, обе оси в gate-доках, правило в CLAUDE.md, ретро-сноска
   в R13-9/WAVE_SUMMARY).
2. **S4 — сквозная пропускная способность drain-пути:** следующий уровень —
   не маршрутизация сканов, а объём drain-работы (батчинг реклейма,
   амортизация `sync_directory_for_segment_classes` по сегменту); сначала
   инструментировать, куда уходят ~17 ms. → **влито в #288 п.6** как
   инструментовка + зафиксировано здесь как отдельное направление Round 15,
   если инструментовка покажет крупный резерв.
3. **S5 — recovery вместо/помимо recycle-reset:** пере-материализация
   sidecar с ре-синком вернула бы 21× heap'ам, поймавшим транзиентный OOM
   (сильнее, чем только сброс на recycle из ревью A). → **влито в #287**
   как второй дизайн-вариант, оценить оба.
4. **§1.3/§4.2 — дрейф абсолютных ns README-таблицы ±60%** между запусками
   у всех трёх аллокаторов одновременно; нормативны только соотношения;
   предложены калибровочный якорь хоста / пометка в шапке / «абсолюты
   только из iai». → **влито в #295 п.6**.
5. **§4.1 — `cargo hack check --feature-powerset --depth 2`** как дешёвый
   структурный ответ на R13-12-класс (E0599 на нехоженой комбинации). →
   **уточнение в #295 п.5**.
6. **§4.4 — вес raw-логов:** ~35 файлов за волну, два полных bench-table
   лога по ~1935 строк; дополнить policy усечением цитируемого лога до
   релевантных секций с пометкой. → **влито в #295 п.7**.
7. **§4.5 — гигиена:** диапазон волны фиксировать после финального
   push/amend (`cb24266` недостижим, заменён `c7e7a79`);
   `2026-07-23-r14-plan.md` не закоммичен. → **влито в #295 п.8**.
8. **§3 P3 — мелочи:** `const _: () = assert!(TCACHE_CAP <= 16)` рядом с
   `virgin_mask` (маска u16 ⟺ CAP≤16 сейчас держится на соглашении);
   название «A/B/B/A» в WAVE_SUMMARY слегка переобещает протокол. →

## 3. Что ревью A добавило, ранее не разнесённое по задачам

   формулировки).

   микродрейф `alloc-stats` против plain-сиблинга. → **влито в #295 п.9**.
