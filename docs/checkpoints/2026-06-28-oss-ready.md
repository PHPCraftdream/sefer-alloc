## Decisions

- **oh-agent's P0 review treated as authoritative.** All 3 P0 defects verified
  by hand (grep allow(unsafe_code) src/ → 13 files; perf numbers
  in MALLOC_BENCH ~45ns/~718ns/16× vs README old 42/788/18x; ls
  benches/*.rs | wc -l = 10) and fixed in 6a71fbb. Source of truth:
  src/lib.rs doc-comment (corrected — added the missing numa.rs).
- **MALLOC_BENCH.md = source of truth for perf numbers.** README/CHANGELOG
  aligned to it (more conservative). Alternative (update
  MALLOC_BENCH to match my numbers) rejected — agent's numbers are more
  recent/repeatable.
- **forbid(unsafe_code) headline clarified.** Previously stated
  unconditionally — now honest: default = forbid, allocator-features
  = deny. This is a nuance, but critical for grep-testing.
- **OPT-E pages stay committed in cache** (no decommit/recommit) —
  this eliminated recommit cost on the cache-hit path and gave 5,270× instead of
  the expected 100x. Trade-off: up to 128 MiB RSS hold (= 2 slots x 64 MiB
  cap), documented.
- **P1/P2 recommendations from the oh-agent deferred** until the user's explicit
  decision — slotmap/jemalloc comparisons and heap_differential.rs
  mention are not publication blockers.
